## Documentation Writer Playbook

### Documentation Types

| Type         | Location          | Format                         |
| ------------ | ----------------- | ------------------------------ |
| Project docs | `content/docs/`   | Markdown (Nuxt Content)        |
| API docs     | Generated by Kubb | TypeScript types/Zod           |
| Context docs | `.context/docs/`  | Markdown with YAML frontmatter |

### Writing Style

- Use Portuguese for descriptions when matching project convention
- Code comments in English
- Tables for structured information (endpoints, types, commands)
- ASCII diagrams for architecture flows
- Keep concise — developers scan docs, they don't read novels

### Nuxt Content Pages

Content files live in `content/docs/` as Markdown with YAML frontmatter:

```markdown
---
title: Page Title
description: Brief description
---

# Page Title

Content here...
```

### Verification

- Ensure all links are relative and valid
- Code examples must follow project style (no semicolons, single quotes)
- Check that documented commands match `package.json` scripts
